# 横断サマリー（新・分析画面：サマリカード）仕様書

最終更新: 2026-04-14

## 0. 背景と目的

KPIツリーを「仮説（目的→要因の構造）」として扱い、**ツリーから自動生成される複数のサマリカード（時系列表）を1画面で俯瞰**できるようにする。

- KPIツリー作成（既存）→ サマリカード生成（今回）→ 分析（回帰など：次フェーズ）
- 本フェーズは「分析」ボタンを配置するが、**中身は次フェーズで実装**する（ダミー）

## 1. 画面仕様

### 1.1 画面の場所

横断サマリー配下に **新しい分析画面**として追加する想定。

- 例: `/projects/[projectId]/unified-summary/analysis`（パス名は実装時に確定）

### 1.2 画面レイアウト

上部に共通コントロール、下部にカード一覧を表示する。

- **ヘッダー操作部**
  - KPIツリー選択（複数ツリー対応）
  - 集計粒度 `timeUnit`: `day | week | month`
  - 分析期間 `dateRange`: `YYYY-MM-DD`〜`YYYY-MM-DD`（両端含む）
- **メイン**
  - KPIツリーから生成された **サマリカード群**
  - 表示は「縦並び」または「2列グリッド」（初期はどちらか固定でOK）

## 2. サマリカード仕様（最重要）

### 2.1 カード生成単位

**親ノードごとに1カード**を生成する。

- 「子を1つ以上持つノード」＝カード対象

### 2.2 カードタイトル

- 親ノードの `label`

### 2.3 カード内の表（時系列テーブル）

- **横軸（列）**: `timeUnit` と `dateRange` から生成される期間ラベル
- **縦軸（行）**: 親子関係に基づく指標行

#### 2.3.1 行の内訳（Y/Xの考え方）

- **Y（目的）行**: 親ノードが `metricRef` を持つ場合のみ表示
- **X（要因）行**: 親ノードの **直下の子ノード群**を表示

親が folder かつ `metricRef` が無い場合でも **カードは生成する**。

- この場合は「タイトル＋子の表のみ」とし、Y行は表示しない

#### 2.3.2 子が folder の場合（展開ルール）

子が folder の場合は **leaf まで展開**して表示する。

- folder 自体は「見出し行（グループ行）」として表示してよい
- **leaf数が多い folder はデフォルト折りたたみ**
  - ユーザーが展開して leaf を確認できる

> 本フェーズでは folder を「1行の集計指標」として扱わない（sum/avg 等の集計定義が必要になるため）。

### 2.4 並び順（確定）

- **カード順（親ノード）**: ツリー順（上から）
- **カード内の行順**
  - Y行（親が `metricRef` を持つ場合）は先頭固定
  - 子（X）はツリー順
  - folder展開したleafもツリーの順序でフラット化（自然順）

### 2.5 行数が多い場合の表示（性能・UX）

- 1カードの初期表示は **最大50行**
- 51行目以降は「さらに表示（+50）」の段階表示（ページ番号によるページングはしない）

### 2.6 右上の「分析」ボタン（次フェーズへの布石）

各カード右上に「分析」ボタンを設置する。

- 本フェーズではダミー（クリックで案内を表示）
- 次フェーズで「このカード（親子セット）」を対象に分析（回帰など）を実行する

## 3. 期間・集計仕様

### 3.1 入力

- 集計粒度 `timeUnit`: `day | week | month`
- 分析期間 `dateRange`: `from`〜`to`（両端含む）

### 3.2 週次の区切り（確定）

- **月曜開始（ISO週）**

### 3.3 端の半端期間

`dateRange` にかかる **部分週 / 部分月も含めて表示**する（切り捨てない）。

## 4. 概念データモデル（実装のための共通言語）

### 4.1 KPIノード（既存概念）

代表項目:

- `id`
- `parentId`
- `sortOrder`
- `label`
- `nodeType: folder | leaf`
- `metricRef?: string | null`
- `serviceId?: string | null`

### 4.2 サマリカード（新規概念）

- `cardId`（親ノードIDベースで可）
- `title`（親label）
- `yRef?`（親が `metricRef` を持つ場合のみ）
- `rows[]`（表示行）
  - `rowId`（ノードID）
  - `label`
  - `ref?`（値取得のキー。例: `serviceId::metricRef` など）
  - `isGroupHeader`（folder見出し行か）
  - `collapsedByDefault`（折りたたみ初期値）
- `periods[]`（列ラベル配列）

## 5. 非ゴール（本フェーズではやらない）

- 「分析」ボタンの実処理（回帰・相関・重み更新等）
- folderノードを自動集計（sum/avg等）して1行の指標として扱う機能
- カードの保存・共有・テンプレ化（必要なら別途設計）
