# 横断サマリー（新・分析画面：サマリカード）実装計画書

最終更新: 2026-04-14

## 0. ゴール

仕様書 `docs/unified-summary-summary-cards-spec.md` の内容を満たす。

- KPIツリーから、親ノード単位でサマリカードを生成
- 1画面に複数カードを表示（縦 or 2列）
- day/week/month + dateRange に対応（週は月曜開始）
- 子folderはleafまで展開し、leaf多い場合は折りたたみ
- 1カード最大50行 + 「さらに表示（+50）」
- 右上に「分析」ボタン（ダミー）

## 1. 前提・依存（着手前に確認）

### 1.1 DBマイグレーション適用

分析画面はKPIツリーやカスタム指標に依存するため、まずDBスキーマが揃っていることを確認する。

- `supabase/migrations/032_kpi_trees.sql`
- `supabase/migrations/033_custom_metrics.sql`

### 1.2 API疎通

最低限、以下が 200 を返し、必要データが取得できること。

- KPIツリー一覧/取得（`/api/projects/[projectId]/kpi-trees` など）
- KPIノード一覧（`/api/projects/[projectId]/kpi-tree/nodes`）

## 2. 実装ステップ

### Step A: 画面ルーティングの追加

- 既存の横断サマリー配下に新ページを作成
  - 例: `src/app/(dashboard)/projects/[projectId]/unified-summary/analysis/page.tsx`

成果物:
- 新ページが表示できる（固定文言でもOK）

### Step B: 画面ヘッダー（入力UI）

実装する入力:
- KPIツリー選択
- `timeUnit` 選択（day/week/month）
- `dateRange` 入力（from/to）

成果物:
- 画面内 state として保持できる
- URLクエリ同期（必要なら。後回しでも可）

### Step C: KPIツリー取得とカード生成ロジック

1) KPIノード取得（API）  
2) ノード配列 → サマリカード配列へ変換

カード生成ルール（仕様の実装）:
- 「子を持つノード」＝カード対象
- カードタイトル＝親label
- 親にmetricRefがあればY行を先頭に入れる
- 子は直下の子をX行に入れる
- 子がfolderならleafまで展開
  - folder見出し行を入れる
  - leaf数が多いfolderは collapsedByDefault=true
- 並び順はsortOrderベース（ツリー順）

成果物:
- ダミーデータではなく、実データのツリーから cards が生成される

### Step D: 期間生成（day/week/month + dateRange）

期間生成ユーティリティを実装/再利用し、periods（列）を生成する。

- day: 日付キー配列
- week: **月曜開始（ISO）** の週キー配列（部分週も含む）
- month: 月キー配列（部分月も含む）

成果物:
- periods が正しい（テスト/ログで検証）

### Step E: 指標値の取得（既存資産の再利用）

基本方針:
- 既存の横断サマリが持つ「指標を period ごとの値にする仕組み」を再利用する
- カード rows の ref（例: `serviceId::metricRef`）から値を引き、テーブル行にする

成果物:
- カード内テーブルに値が埋まり、欠損は `null` → 表示は `—`

### Step F: カードUI（表 + 折りたたみ + 段階表示）

実装内容:
- カードの枠、タイトル、右上ボタン（ダミー）
- 表（横スクロールあり）
- folder見出し行の折りたたみ（デフォルトは条件に応じて閉）
- 初期50行、追加表示（+50）で段階表示

成果物:
- 大きいツリーでも最低限の操作性が担保される

### Step G: 受け入れ確認（チェックリスト）

以下が満たされること:
- 1つのKPIツリーから複数カードが生成され、1画面に表示される
- 親がfolderかつmetricRefなしでもカードが出る（Y行なし）
- 子folderは折りたたまれ、展開でleafが出る
- day/week/month + dateRange で列が切り替わる
- 週次は月曜開始
- 50行制限と「さらに表示」が動作する
- 「分析」ボタンは表示され、押すとダミー挙動

## 3. リスクと対策

- **カード行数・期間列数が多いと重い**
  - 対策: 50行制限、折りたたみ、必要なら期間上限（例: 最大365日）を設ける
- **folderノードが多段で深いと読みにくい**
  - 対策: 見出し行 + 折りたたみ、インデント/バッジで階層を視覚化
- **週/月集計の境界がユーザー期待とズレる**
  - 対策: ISO週（月曜開始）を明記、UIに集計ルールのヘルプを出す

## 4. 次フェーズ（スコープ外だが接続点）

- 各カード右上の「分析」ボタン
  - このカード（親Y + 子X）のセットを入力に回帰を実行
  - 結果の保存、比較、全体分析（上位親への連鎖）へ拡張
